Vulnerability of the Day is a pedagogically-curated collection of vulnerability demonstrations for undergraduate software engineering students. The goal is to show how to avoid simple coding mistakes by providing concise code examples. Key characteristics are:

  • 10 minutes long or less
  • Targeted to third-year college students in a software engineering course
  • Socially-relevant domains
  • Can be run on a Linux console, using make
  • Concise without being contrived
  • Real-world historical exmaples with source code patches linked
  • Exhibiting a wide variety of key security principles

Check out the code on GitHub and the Catalog on the wiki!
